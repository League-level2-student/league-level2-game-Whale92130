Apple Fall

by: Alexander Sementchenko

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I wanted to make this game because it was a simple enough game that is fun to play. It also had many possibilities to add features to make it more interesting.

2. What I Learned

I learned from user testing that my game had a lot of bugs that I had not fixed. I also got feedback to add difficulty modes and add more power ups.

3. Architecture of My Game

The game has many different parts that all interact together. The GamePanel controls what is on the screen while getting information from the ObjectManager which controls all the objects in the game, the basket, apples, bombs, and powerups. The GameObject is a super class for all the objects. The AppleFall class runs the whole thing and has the main method.

4. Technical Implementation

A technical part of my game was making the game get harder over time and to be able to change the difficulty in the menu. The steps it took to create this were to create a new timer for the apple spawn rate whenever an apple was collected. This new timer would have a shorter time than the previous one. It would decrease by a certain amount based on the mode. The timer would stop decreasing after reaching a 500ms delay. The timers delay would have the score be multiplied by the difficulty value(30,100,500).

5. Most Challenging Part

The most challenging part of this project was making making the difficulty change over time. I tried different methods to adjust the difficulty in real time until a solution worked.

6. Future Fixes and Updates

I would add more power ups and refine the movement. I would also add some easter eggs.
